
《秩序之眼》Demo 设计文档

═══════════════════════════════════════════════════════
一、 操作规则与核心机制
═══════════════════════════════════════════════════════

1. 基础操控
   [WASD]        移动
   [鼠标]        视角旋转
   [空格]        跳跃（消耗 15 体力，0.12s 土狼时间宽容）
   [鼠标左键]    射击（单击单发，射速 0.2s/发，弹速 150m/s）
   [鼠标右键]    闪避（向后纯水平冲刺，消耗 25 体力，0.5s 无敌帧）
   [按住 Shift]  开启秩序之眼（定身观察）

2. 闪避输入缓冲
   0.15s 缓冲窗口：在攻击命中前 0.15s 内按下右键，即使当前帧不满足
   闪避条件（如正在其他动作中），系统也会在下一个可执行帧自动补发闪避。
   目的：消除高节奏战斗中的"输入吞没"感，保证操作响应性。

3. 体力系统
   最大体力 100，自然恢复速率 10/s（闪避期间停止恢复）。
   体力条以弧形点阵显示在玩家右侧（-60° 至 +60° 极坐标），
   黄色点阵 + 黑色描边，30% 以下变红预警。
   视觉位置：跟随玩家世界坐标 + 摄像机右侧 0.4m 偏移。

4. 秩序之眼（Shift）机制
   按住 Shift 期间：
   - 玩家完全定身（无法移动/闪避/射击）
   - 敌人红色外壳隐去，暴露黄色能量核心
   - 敌人攻击预警区域（黄色几何体）变得可见
   - 飞弹（红色方块）变得可见
   - 震地冲击波圆环变得可见
   - 攻击预警音效得以播放（秩序隔离：不按 Shift 时仅子弹时间/防反Buff期间可听）
   松开 Shift 后一切恢复正常。
   核心博弈：牺牲机动性换取信息优势。

5. 射击与伤害结算
   子弹为 0.05×0.05×0.2 的发光青色立方体。
   命中敌人的伤害根据当前状态结算：
   ┌─────────────┬──────────────────────────────────────┐
   │ 敌人状态     │ 效果                                  │
   ├─────────────┼──────────────────────────────────────┤
   │ Normal      │ 伤害 ×0.1（外壳防护，几乎无伤）       │
   │ Telegraphing│ 伤害 ×0.1（外壳防护，几乎无伤）       │
   │ Recovery    │ 全额伤害（抓后摇窗口）                 │
   │ Recovery+Buff│ 触发防反破核 → Boss 瘫痪 5 秒         │
   │ Stunned     │ 伤害 ×2.0（核心暴击）                  │
   └─────────────┴──────────────────────────────────────┘

6. 完美闪避与子弹时间
   在无敌帧（0.5s）内被攻击命中 → 触发完美闪避：
   - 画面白色扩散光圈反馈
   - 子弹时间：Time.timeScale = 0.2，持续 0.5s（真实时间）
   - 获得 2 秒"防反破核 Buff"
   - Buff 期间射击 Recovery 状态的 Boss → 立即瘫痪
   - GameLogger 输出 "完美闪避！触发子弹时间！"

7. 生命与死亡
   ── 玩家死亡 ──
   玩家生命值 100，受到攻击扣减。
   生命值归零 → Die()：
   - Time.timeScale = 0（全场时停）
   - 屏幕灰色半透明遮罩
   - 大字"卡 了"（卡了）
   - "重新开始"按钮（重载当前场景）
   - "返回菜单"按钮（返回 Index 0）
   - 死亡状态下 Esc 暂停菜单被禁用

   ── Boss 死亡与胜利 ──
   Boss 血量归零 → EnemyOrderSystem.Die()：
   - 停止攻击循环与瘫痪计时器
   - 销毁所有在场飞弹
   - 3 个 Drone 各自触发碎裂特效（SpawnDeathEffect）：
     在原位生成 10 块红色立方体碎片，施加爆炸力向外飞散，2 秒后自动销毁
   - Boss GameObject 销毁（Destroy）
   - VictoryManager 触发胜利界面：
     Time.timeScale = 0、全屏半透明遮罩、居中"胜利"标题、
     "重新开始"按钮（重载当前场景）/ "返回菜单"按钮

8. 暂停菜单
   Esc 键呼出暂停菜单：
   - 继续游戏 / 重新开始 / 退出游戏
   - 全屏黑色半透明遮罩 + 居中暗色面板
   - 玩家死亡时无法呼出

二、 背景设定与攻略
═══════════════════════════════════════════════════════

【背景设定】
2077年，物理学崩坏，世界陷入了"无序态"。
所有物质都在疯狂坍缩，肉眼所见皆是致命的混沌。
唯有植入"秩序之眼"的躯壳，才能在毁灭的洪流中，
捕捉到那一丝名为"真相"的蓝光。
在秩序崩坏的废墟上，你的眼，即是最后的准星。

玩家扮演一名植入"秩序之眼"的猎手，对抗由混沌凝聚而成的有序实体。
未开启秩序之眼时：玩家视野中没有任何声音、预警、特效，只能被无序攻击无厘头地命中并掉血。
开启秩序之眼后：玩家可观察到有序的攻击轨迹——有预警、有声音、有特效，但代价是完全定身。
因此，玩家必须先在秩序态下观察并记忆攻击模式，再关闭秩序之眼，凭记忆进行闪避与反击。

【Boss 攻略】

核心循环：观察 → 闪避 → 抓后摇 → 输出

第一步：观察前摇
  按住 Shift 进入秩序视野，观察 Boss 的黄色预警区域，蓄力黄点及核心位置。
  四种招式的预警各不相同，玩家需要在 1.2s 内判断攻击类型。

第二步：精确闪避
  在攻击命中瞬间按下鼠标右键触发闪避。
  利用 0.15s 输入缓冲 + 0.5s 无敌帧覆盖攻击判定窗口。
  成功闪避 → 触发子弹时间 + 2s 破核 Buff。

第三步：抓后摇输出
  Boss 攻击结束后进入 1.5s Recovery状态。
  此时射击可造成全额伤害。
  如果持有防反 Buff，射击 Recovery 状态的 Boss 将触发"防反破核"。

第四步：核心暴击
  Boss 被防反破核后瘫痪 5 秒，核心暴露。
  此时射击伤害 ×2.0，是最佳输出窗口。

三、 设计思路
═══════════════════════════════════════════════════════

1. 核心设计哲学：信息不对称博弈
   本作的核心体验来源于"秩序"与"无序"两种状态之间的信息差。
   无序态下玩家拥有全部机动性，但看不到攻击预警，只能凭直觉反应。
   秩序态下玩家能看到全部预警，但失去一切机动性，成为静止靶子。
   这种"安全与信息不可兼得"的矛盾构成了持续的决策压力。

2. 三体幻影与核心轮转
   Boss 拆分为 3 个外观相同的 Drone，引入"真假身"识别层。
   玩家必须在持 Shift 观察（定身）与松 Shift 移动（失明）之间
   快速切换，同时完成：识别真身 → 闪避攻击 → 抓后摇输出。
   攻击权与真身分离的设计进一步增加决策复杂度：
   攻击你的 Drone 可能是假身，真身可能在远处静默等待。

3. 参考与拆解
  ── 游戏参考──
   - 异环（完美世界游戏）：极限闪避触发子弹时间、精准弹反（黄色提示圈）。
     本作的完美闪避 + Witch Time 直接借鉴此设计思路。
   - 原神（米哈游）：Boss 前摇预警 + 闪避无敌帧（0.3s）机制。
     本作将无敌帧延长至 0.5s，并加入输入缓冲（0.15s）进一步降低门槛。
     原神深渊 Boss 的"分身/多阶段"设计启发了本作的三体幻影机制。

── acg/机制参考 ──

Re:从零开始的异世界生活（白鲸讨伐战）：
核心参考其“分身机制”与“战局反转结构”。白鲸在受到致命威胁后，通过分裂与高压环境重构战场，使战斗从“可控”瞬间转变为“混乱与失控”。
本作将该设计抽象为 Phase Transition 机制：在 Boss 血量过半时，通过强制演出完成“单体 → 多体”的结构跃迁，打断玩家已建立的节奏与安全感。
白鲸战中的“信息压迫与绝望感构建”：
原作通过多目标干扰与战场混乱提升观感压力。本作在 P2 阶段引入三体独立 AI 与交叉攻击路径，将原本清晰的单目标预判转化为多源威胁场，显著提升玩家的视觉负荷与决策成本。
战斗节奏与心理落差设计：
参考白鲸战“前期试探 → 中期掌控 → 后期崩塌”的节奏变化，本作在 P1 阶段提供稳定的招式学习环境，在阶段切换后通过机制复杂度跃迁制造“节奏断裂”，强化战斗张力与记忆点。
── 音效参考 ──
   - CS 系列（Valve）：爆头音效与枪声的听觉反馈设计。
     本作的射击音效参考了 CS 中清脆、短促、有辨识度的枪声设计理念，
     追求"每发子弹都有重量感"的听觉反馈。
   - 全部音效素材来源：freesound.org（经个人剪辑与调速修改）。
4. 难度设计思路
   设计上预留了难度扩展：
   - Boss 1000 HP + 四种攻击随机切换 → 基础难度
   - 可通过调整 waitDuration（出招间隔）缩短来增加压力
   - 可通过增加 missileCount（飞弹数量）来提高反应要求
   - 可通过减小 chargeHalfWidth（冲锋宽度）来提高闪避精度要求
   数据结构已分离为 Inspector 可调参数，支持快速迭代。

四、 BOSS AI：三体幻影（Eye of Order）
═══════════════════════════════════════════════════════

3 个红色立方体 Drone 共享 1000 HP，招式数值基础值 × attackScale(1.8)。

【双阶段结构】
P1 巨物压迫：仅激活 Drone_0（3 倍体型），独占全部攻击，玩家专注学习招式。
P2 三体幻影：50% HP 触发分裂演出（冻结 1s → 缩放 → 激活双体 → 解冻），
  3 体独立巡逻（走 2~3s / 停 1~2s），超出 12m 或贴近 3m 时 120° 折返，
  间距 < 3m 同伴避让，Y 轴锁定 0.5f，始终 LookAt 玩家。

【核心轮转与攻击权分离】
P1 恒定 Drone_0 为真身；P2 每次前摇随机更换真身（currentCoreIndex）。
真身与攻击者分离——攻击你的 Drone 未必是真身。
- 无序态：3 体外观完全相同（纯红实体）
- 秩序态（Shift/子弹时间/防反Buff）：仅真身变为半透明，暴露黄色核心
- Stunned（瘫痪 5s）：全部灰色半透明，核心外露

【伤害结算】
命中需同时满足：(1) 打中真身 (2) CanPierceCore 生效，否则免疫。
命中后 0.1s 受击冷却防抖，免疫不触发冷却。
状态机：Normal → Telegraphing → Recovery → Normal；防反破核 → Stunned(5s)。
出招间隔 3s / 蓄力 1.5s / 后摇 1.5s，前 0.3s 亮黄色预警球。
预警音效仅 CanPierceCore 时可听（秩序隔离）。

【四招招式】
招式 0 — 直线冲锋：2m×8m 贴地矩形预警，0.3s 瞬移，伤害 25
招式 1 — 扇形横扫：程序化扇形 Mesh，EaseIn 砸落，伤害 20
招式 2 — 震地冲击波：空心圆环扩散至 8m，环内安全，伤害 35
招式 3 — 贝塞尔飞弹：3 枚追踪弧线弹，锁定后发射，伤害 15/枚
  ※ 扇形横扫仅在距玩家较近时触发，远处替换为冲锋/冲击波。
