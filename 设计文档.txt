
《秩序之眼》Demo 设计文档

═══════════════════════════════════════════════════════
一、A  操作规则
═══════════════════════════════════════════════════════

1. 基础操控
   [WASD]        移动（前/后/左/右）
   [鼠标]        视角旋转（水平转向 + 垂直俯仰 ±60°）
   [空格]        跳跃（消耗 15 体力，0.12s 土狼时间宽容）
   [鼠标左键]    射击（射速 0.2s/发，弹速 150m/s）
   [鼠标右键]    闪避（向后冲刺，消耗 20 体力，获得 0.5s 无敌帧）
   [按住 Shift]  开启秩序之眼（定身观察）

2. 闪避输入缓冲
   0.15s 缓冲窗口：在攻击命中前 0.15s 内按下右键，即使当前帧不满足
   闪避条件（如正在其他动作中），系统也会在下一个可执行帧自动补发闪避。
   目的：消除高节奏战斗中的"输入吞没"感，保证操作响应性。

3. 体力系统
   最大体力 100，自然恢复速率 10/s（闪避期间停止恢复）。
   体力条以弧形点阵显示在玩家右侧（-60° 至 +60° 极坐标），
   黄色点阵 + 黑色描边，30% 以下变红预警。
   视觉位置：跟随玩家世界坐标 + 摄像机右侧 0.4m 偏移。

4. 秩序之眼（Shift）机制
   按住 Shift 期间：
   - 玩家完全定身（无法移动/闪避/射击）
   - 敌人红色外壳隐去，暴露蓝色能量核心
   - 敌人攻击预警区域（黄色几何体）变得可见
   - 飞弹（红色方块）变得可见
   - 震地冲击波圆环变得可见
   松开 Shift 后一切恢复正常。
   核心博弈：牺牲机动性换取信息优势。

5. 射击与伤害结算
   子弹为 0.05×0.05×0.2 的发光青色立方体。
   命中敌人的伤害根据当前状态结算：
   ┌─────────────┬──────────────────────────────────────┐
   │ 敌人状态     │ 效果                                  │
   ├─────────────┼──────────────────────────────────────┤
   │ Normal      │ 伤害 ×0.1（外壳防护，几乎无伤）       │
   │ Telegraphing│ 伤害 ×0.1（外壳防护，几乎无伤）       │
   │ Recovery    │ 全额伤害（抓后摇窗口）                 │
   │ Recovery+Buff│ 触发防反破核 → Boss 瘫痪 5 秒         │
   │ Stunned     │ 伤害 ×2.0（核心暴击）                  │
   └─────────────┴──────────────────────────────────────┘

6. 完美闪避与子弹时间
   在无敌帧（0.5s）内被攻击命中 → 触发完美闪避：
   - 画面白色扩散光圈反馈
   - 子弹时间：Time.timeScale = 0.2，持续 0.5s（真实时间）
   - 获得 2 秒"防反破核 Buff"
   - Buff 期间射击 Recovery 状态的 Boss → 立即瘫痪
   - GameLogger 输出 "完美闪避！触发子弹时间！"

7. 生命与死亡
   玩家生命值 100，受到攻击扣减。
   生命值归零 → Die()：
   - Time.timeScale = 0（全场时停）
   - 屏幕灰色半透明遮罩
   - 大字"卡 了"（红色 120pt，Meta 元素）
   - "重新开始"按钮（重载当前场景）
   - "返回菜单"按钮（返回 Index 0）
   - 死亡状态下 Esc 暂停菜单被禁用

8. 暂停菜单
   Esc 键呼出暂停菜单：
   - 继续游戏 / 重新开始 / 退出游戏
   - 全屏黑色半透明遮罩 + 居中暗色面板
   - 玩家死亡时无法呼出

一、B  背景设定与攻略
═══════════════════════════════════════════════════════

【背景设定】
2077年，世界陷入了"无序态"。
所有物质都在疯狂坍缩，肉眼所见皆是致命的混沌。
唯有植入"秩序之眼"的躯壳，才能在毁灭的洪流中，
捕捉到那一丝名为"真相"的蓝光。

玩家扮演一名被"秩序之眼"改造的猎手，深入无序裂隙深处，
对抗由混沌凝聚而成的巨型实体——"Eye of Order"。

【Boss 攻略】

核心循环：观察 → 闪避 → 抓后摇 → 输出

第一步：观察前摇
  按住 Shift 进入秩序视野，观察 Boss 的黄色预警区域和蓄力黄点。
  四种招式的预警各不相同，玩家需要在 1.2s 内判断攻击类型。

第二步：精确闪避
  在攻击命中瞬间按下鼠标右键触发闪避。
  利用 0.15s 输入缓冲 + 0.5s 无敌帧覆盖攻击判定窗口。
  成功闪避 → 触发子弹时间 + 2s 防反 Buff。

第三步：抓后摇输出
  Boss 攻击结束后进入 1.5s Recovery（后摇）状态。
  此时射击可造成全额伤害。
  如果持有防反 Buff，射击 Recovery 状态的 Boss 将触发"防反破核"。

第四步：核心暴击
  Boss 被防反破核后瘫痪 5 秒（Stunned），核心暴露。
  此时射击伤害 ×2.0，是最佳输出窗口。

各招式应对策略：
  ┌──────────────┬─────────────────────────────────────────────┐
  │ 直线冲锋      │ 横向移动或向后闪避，2m 宽度需提前侧移       │
  │ 扇形横扫      │ 向后闪避脱离 4.5m 半径 + 180° 扇面          │
  │ 震地冲击波    │ 跳跃（空格）越过 1.5m 宽的扩散空心环         │
  │ 贝塞尔飞弹    │ 闪避穿过飞弹路径，利用无敌帧触发完美闪避     │
  └──────────────┴─────────────────────────────────────────────┘

一、C  设计思路
═══════════════════════════════════════════════════════

1. 核心设计哲学：信息不对称博弈
   本作的核心体验来源于"秩序"与"无序"两种状态之间的信息差。
   无序态下玩家拥有全部机动性，但看不到攻击预警，只能凭直觉反应。
   秩序态下玩家能看到全部预警，但失去一切机动性，成为静止靶子。
   这种"安全与信息不可兼得"的矛盾构成了持续的决策压力。

2. 三体幻影与核心轮转
   Boss 拆分为 3 个外观相同的 Drone，引入"真假身"识别层。
   玩家必须在持 Shift 观察（定身）与松 Shift 移动（失明）之间
   快速切换，同时完成：识别真身 → 闪避攻击 → 抓后摇输出。
   攻击权与真身分离的设计进一步增加决策复杂度：
   攻击你的 Drone 可能是假身，真身可能在远处静默等待。

3. 参考与拆解
   - 猎天使魔女（Bayonetta）：完美闪避触发子弹时间 → 本作的 Witch Time
     系统直接借鉴此机制，但将判定窗口放宽至 0.5s（原作约 0.2s），
     降低硬核门槛以匹配技术策划笔试的可玩性要求。
   - 黑暗之魂（Dark Souls）：Boss 招式前摇 → 攻击 → 后摇的节奏循环。
     本作将四种攻击几何化，每种都需要不同的空间应对策略。
   - 鬼泣（Devil May Cry）：华丽度评价系统。本作简化为状态判定表
     （刮痧/全额/暴击），用状态机而非连招驱动输出决策。

4. 几何判定 vs 传统碰撞体
   所有 Boss 攻击均采用数学运算而非物理碰撞体：
   - 冲锋：Vector3.Dot + Vector3.Cross 实现 OBB 矩形投影判定
   - 横扫：Vector3.Angle + magnitude 的极坐标扇形判定
   - 冲击波：动态 innerR/outerR 的空心环判定，Mesh 顶点与判定共享同一变量
   - 飞弹：Physics.OverlapSphere 半径 1.5m 的体积判定
   优势：视觉与判定严格同步（误差 = 0），避免物理引擎的不确定行为。

5. 程序化 Mesh 驱动视觉
   圆环冲击波和扇形横扫的视觉体均通过代码生成 Mesh 顶点。
   圆环使用 BuildRingMesh() 构建 64 段双层圆环，每帧 UpdateRingMesh()
   原地更新顶点坐标，GameObject localScale 始终为 (1,1,1)，
   彻底消除 scale 换算误差。
   扇形使用 BuildFanMesh() 构建 48 段三角面，顶点半径 = 判定半径。

6. 纯 OnGUI 渲染方案
   全部 UI（准星、血条、体力条、Boss 血条、死亡界面、暂停菜单、
   控制台日志）均使用 OnGUI 即时模式渲染。
   原因：无需 Canvas/Prefab/AssetBundle，纯代码零资源依赖，
   符合"简单几何体搭建"的题目要求。

7. Shader 容错设计
   材质系统采用 URP → Standard 三级降级：
   - 首选 Universal Render Pipeline/Lit
   - 降级 Standard shader
   - 无光照层使用 Unlit/Color、Sprites/Default
   保证在任意渲染管线配置下均可运行。

8. 难度设计思路
   虽然题目选项 B 只要求一个对抗因子，但设计上预留了难度扩展：
   - Boss 1000 HP + 四种攻击随机切换 → 基础难度
   - 可通过调整 waitDuration（出招间隔）缩短来增加压力
   - 可通过增加 missileCount（飞弹数量）来提高反应要求
   - 可通过减小 chargeHalfWidth（冲锋宽度）来提高闪避精度要求
   数据结构已分离为 Inspector 可调参数，支持快速迭代。

二、 BOSS AI 系统：Eye of Order 5.0（三体幻影）
═══════════════════════════════════════════════════════

Boss 由 3 个红色立方体 Drone 组成，以玩家为中心、半径 8m、
各 120° 间隔在 XZ 平面上缓慢环绕漂浮（10°/s）。
3 个 Drone 共享同一血条（1000 HP）。

── 核心轮转（Core Roulette）──
每隔 5~8 秒，系统随机选取一个 Drone 成为"真身"（currentCoreIndex）。
- 无序态（不按 Shift）：3 个 Drone 外表完全相同（纯红色实体）。
- 秩序态（按住 Shift）：只有真身 Drone 变为半透明，内部暴露黄色核心；
  另外两个 Drone 依旧保持红色实体外观。
- Stunned（瘫痪）：全部 3 个 Drone 变为灰色半透明，黄色核心全部暴露。

── 伤害免疫系统 ──
玩家射击命中某个 Drone 时，必须同时满足：
  1. 命中目标的索引 == currentCoreIndex（打对真身）
  2. isOrder == true（必须按住 Shift 看破）
否则伤害强制为 0，GameLogger 输出"幻影免疫！"或"攻击无效！"。

── 攻击权分离（Attack Separation）──
真身 Drone 不一定是发动攻击的那个！
每次攻击前，系统随机抽取一个 Drone 作为 currentAttacker。
所有攻击预警（黄点、几何黄圈）和攻击动作（瞬移、发波）
均由 currentAttacker 执行。

状态机：Normal → Telegraphing → Recovery → Normal（循环）
         ↓ (防反破核)
       Stunned（5s）→ Normal

出招间隔 3s（Normal），蓄力 1.5s（Telegraphing），后摇 1.5s（Recovery）。
每次攻击前 0.3s 会亮起黄色球体渐隐动画作为统一预警信号。

招式 0 —— 直线冲锋（Linear Dash）
  预警：2m×8m 长条矩形（仅 Shift 可见），黄色半透明贴地 Plane。
  判定：OBB 投影法。
    dashDirH = attacker.forward 的 XZ 水平化
    fwdProj  = Dot(toPlayer, dashDirH) ∈ [0, 8]
    perpProj = |Cross(dashDirH, toPlayer).y| ≤ 1.0
  表现：attacker Drone 沿 forward 方向瞬移 8m（0.3s 内 Lerp）。
  伤害：25
  视觉判定同步：Plane.scale.z×10 = chargeLength = 8m,
                 Plane.scale.x×10 = chargeHalfWidth×2 = 2m → 半宽 1m ✓

招式 1 —— 扇形横扫（Fan Sweep）
  预警：程序化扇形 Mesh 贴地（仅 Shift 可见），黄色脉冲。
  判定：magnitude < 4.5m AND Vector3.Angle(attacker.forward, toPlayer) < 90°。
  表现：红色扇形体从 attacker 上方 5m 高空以 EaseIn 加速砸落（0.1s），
        停留 0.3s 后 0.18s 渐隐消散。
  伤害：20
  视觉判定同步：fanMesh 顶点半径 = sweepRadius = 4.5m,
                 fanMesh 弧度 = sweepHalfAngle×2 = 180° ✓

招式 2 —— 震地冲击波（Ground Shockwave）
  预警：从 attacker 中心向外脉冲扩张的黄色圆环（Shift 可见）。
  判定：空心圆环逻辑。
    origin = attacker.position（XZ 平面）
    midRadius = Lerp(0, 8, t)
    innerR = max(0, midRadius - 0.75)
    outerR = midRadius + 0.75
    hit = (dist2D ∈ [innerR, outerR]) AND (player.y < 1.5)
  表现：程序化 Mesh 圆环从 attacker 中心扩散至 8m，透明度渐弱。
  伤害：35
  安全区：attacker 中心（环未到达）或 8m 外（环已扫过）。
  应对：跳跃（空格）离开地面即可规避。
  视觉判定同步：Mesh 顶点与判定共享 innerR/outerR 变量 ✓
  圆环 Y = 0.2f（高于地面，防 Z-fighting）。
  圆环 MeshRenderer.enabled 严格跟随 isOrderVision（Shift 绑定）。

招式 3 —— 贝塞尔弧线飞弹（Bézier Projectiles）
  蓄力阶段：追踪预警圈（小黄色圆）实时跟随玩家脚下。
  锁定时刻（-0.3s）：玩家当前位置被冻结为锁定目标。
  发射：3 枚红色立方体飞弹（0.2×0.2×0.5），走不同二次贝塞尔弧线。
    Bézier 公式：Pos(t) = (1-t)²·P0 + 2(1-t)t·P1 + t²·P2
    fireOrigin = attacker.position + Vector3.up * 1.2
    飞弹 0：P1 大幅向上（高弧抛物线，从天而降）
    飞弹 1：P1 右偏 + 上偏（右绕包抄弧线）
    飞弹 2：P1 左偏 + 上偏（左绕夹击弧线）
  飞弹间隔 0.12s 依次发射，单枚飞行时长 0.7s。
  判定：Physics.OverlapSphere 半径 1.5m（大体积配合无敌帧触发完美闪避）。
  视觉：飞弹 MeshRenderer.enabled 严格跟随 Shift 视野。
  伤害：15/枚

═══════════════════════════════════════════════════════
三、 技术系统实现 (纯代码驱动)
═══════════════════════════════════════════════════════

1. 三体幻影系统 (Drone Manager)
   Boss 由 EnemyOrderSystem 统一管理 3 个独立 Drone 实体。
   每个 Drone 包含：红色立方体壳 + 黄色球形核心（Order 可见）。
   OrbitDrones 协程：以玩家为中心、8m 半径、120° 间隔环绕。
   CoreShiftCycle 协程：每 5~8s 随机切换真身索引。
   攻击协程：随机选取 attacker Drone，所有视觉/判定从 attacker 位置发起。
   受击判定：GetDroneIndex(Collider) 识别命中 Drone 索引，
            IsCorrectDrone(idx, isOrder) 判断是否为有效打击。

2. 弧形体力条 (Radial Stamina)
   算法：三角函数计算 -60° 到 +60° 极坐标点阵。
   位置：玩家世界坐标 + 摄像机 right 方向 0.4m 偏移。
   渲染：黑色 6×6 底层描边 + 黄色 4×4 前层点阵（30% 以下变红）。

3. 游戏内控制台 (Game Logger)
   位置：屏幕右上角 300×250 半透明黑底框。
   功能：全局静态 Log 系统，最多保留 10 条记录。
   日志内容：完美闪避、状态切换、Boss 出招、伤害结算、防反破核等。
   替代了所有 Debug.Log 调用。

4. 死亡机制 (Meta-Death)
   触发：玩家生命值 ≤ 0。
   流程：isDead = true → Time.timeScale = 0 → OnGUI 渲染死亡界面。
   UI：灰色半透明全屏遮罩 + "卡 了"（红色 120pt Meta 文字）
       + "重新开始"/"返回菜单"按钮（黑底白字）。
   按钮可见性修复：GUI.contentColor = Color.black 确保文字可见。

5. 碰撞检测方案
   子弹(BulletBehaviour)：Physics.RaycastAll + 按距离排序 + 过滤自身。
   子弹命中 Drone 时，通过 EnemyOrderSystem.GetDroneIndex() 识别目标索引，
   传递给 TakeDamage(dmg, isPerfect, droneIdx) 进行免疫判定。
   飞弹(Bezier)：Physics.OverlapSphere 半径 1.5m（大体积判定）。
   飞弹碰撞过滤：GetDroneIndex(col) >= 0 时跳过（不伤害友军 Drone）。
   所有碰撞均排除友方实体。

6. 蓄力黄点渐隐 (TelegraphFade)
   黄色球体从 scale=0.5 渐缩至 0.15，alpha 从 1.0 渐隐至 0。
   时长 0.3s，位于 Boss 头顶 (0, 1, 0)。
   可见性严格跟随 isOrderVision。

7. 飞弹可见性绑定
   飞弹 MeshRenderer.enabled 在 Update() 中每帧同步 isOrderVision。
   震地冲击波圆环同理，MeshRenderer.enabled 每帧同步 isOrderVision。
   蓄力黄点通过 SetActive 每帧同步 isOrderVision。
   Drone 核心 MeshRenderer.enabled 在 Update() 中每帧同步 isOrderVision。

8. 预警区域 Shift 绑定
   所有招式的预警区域（矩形/扇形/追踪圈/冲击波）均仅在秩序态下可见。
   ApplyVisibility() 在每次状态切换时统一控制所有视觉层的 SetActive。

9. Drone 视觉状态管理
   Update() 中每帧同步 3 个 Drone 的材质：
   - Normal/Recovery：全部 droneChaosMat（纯红色实体）
   - Telegraphing + isOrder：真身 droneOrderMat（半透明），其余 droneChaosMat
   - Stunned：全部 transparentMat（灰色半透明），核心全部暴露
   材质切换通过 Material 引用比较避免重复赋值。

四、 场景流转架构
═══════════════════════════════════════════════════════

Index 0 [MainMenu]
  背景：Resources/MainBG 纹理（16位像素风赛博城市），降级纯色兜底。
  UI：纯 OnGUI —— "秩序之眼" 标题（青色 80pt）+ 开始游戏/退出游戏按钮。
  作者署名右下角。

Index 1 [StoryScene]
  黑屏 + 白色文字从下向上滚动（50px/s）。

═══════════════════════════════════════════════════════
六、 Unity 序列化注意事项
═══════════════════════════════════════════════════════

所有 MonoBehaviour 的 public 字段必须在 Awake() 中强制赋值。
原因：Unity 将 public 字段序列化到场景文件中。当修改代码默认值后，
场景中缓存的旧值会在运行时覆盖代码中的新值。
Awake() 在反序列化之后执行，确保代码中的值始终生效。

当前项目中已在以下脚本的 Awake() 中强制赋值：
  PlayerController    ：dodgeSpeed, dodgeDuration, invincibleDuration
  EnemyOrderSystem    ：全部 24+ 战斗参数
  BulletBehaviour     ：speed, damage, lifeTime
  HomingProjectile    ：speed, homingStrength, damage, lifeTime
